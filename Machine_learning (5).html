Loading the data¶

In today's highly competitive banking industry, retaining existing clients is very important for long-term success of the bank. Customer churn, which refers to the losing of clients over a given period. When it happens the bank not only loses revenue but also is pushed to increase additional costs for new customers. Predicting customers leaving the bank can help a bank identify which customers are most likely to leave so the bank can approach them to try to change it so there are less customers that will leave. It can be done by providing some personal services, giving some deals for them, etc. The business problem that I will be trying to solve in this project is the prediction of customer churn in a bank. My goal is to be able to predict what kind of customers are most likely to leave the bank in the future. To solve this problem, I will use a machine learning approach. By using a dataset from Kaggle:(https://www.kaggle.com/datasets/gauravtopre/bank-customer-churn-dataset). To approach this problem I created a small plan to first visualize the data to make some data exploration, do data preprocessing to prepare data for machine learning and assess the model.

After comparing those 4 models and their accuracy I got results that Decision Trees will be most accurate for my data set so I decided to use it and tune its hyperparameters by performing GridSearch. First I defined the hyperparameter Grid. This implies the hyperparameters that will be tested.

Hyperparameters:

max_depth: Limits the maximum depth of the tree:

3: Max 3 levels.

5: Max 5 levels.

10: Max 10 Levels.

None: No limit tree will keep growing until it cannot be split anymore.

min_samples_split: Minimum number of samples required to split:

2:Node needs 2 samples.

10:Node needs 10 samples.

20: Node needs 20 samples.

min_samples_leaf: Minimum number of samples that a leaf node must have:

1: Leaf node need a minimum of 1 sample.

5: Leaf node need a minimum of 5 samples.

10: Leaf node need a minimum of 10 samples.

After defining hyperparameters I needed to define the model because I decided to use a decision tree i have to define it. After doing that I needed to set up the grid search to find the best hyperparameter combination. GridSearchCV Parameters:

cv=5: 5-fold cross-validation is used.

scoring='accuracy': The metric used to evaluate each model is accuracy.

n_jobs=-1: This is used to tell GridSearchCV to use all CPU available to make sure that program runs faster.
